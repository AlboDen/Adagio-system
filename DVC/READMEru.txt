Программа клиента визуализации данных (далее КВД) являлась первой программой,
написанной мной на питоне. Мой значальный стаж владения данным языком программирования 
составлял около одной недели. Вместе с параллельной разработкой программы 
сервера хранилища данных процесс создания данного ПО составил около 1,5 месяца, пятидневной
рабочей недели.

Техническое задание:
Разработать визуальное клиентское ПО для получения информации о текущих показаниях 
счётчиков водоснабжения. Проработать маршруты пользователя для наиболее удобного 
и понятного использования графического интерфейса. При создании архитектуры оснастить 
ПО как можно большим количеством защит от непредвиденных вводов некорректных значений 
пользователем (в полях, где предусмотрен ввод числовых или иных выражений), обработать 
случаи в которых ПО может выйти из строя и оповестить об этом пользователя (например, 
в случае разрыва интернет-соединения будет высвечиваться поле с желтым знаком восклицания 
и сопровождающая надпись).

В процессе разработки данного ПО были получены навыки:
-  Разработки ПО с привязкой к визуальным формам;
-  Базовое освоение создания архитектуры приложения и блочной структуры кода программы;
-  Базовый этап освоения ООП питона;
-  Были разработаны ключи (первые 5 символов в TCP-пакете) позволяющие идентифицировать
   функционал пакета отсылаемый КВД на сервер хранилища данных для получения требуемого 
   ответа;

Недоработки и баги:
- Нет визуализации ошибки соединения между клиентом сбора информации и клиентской единицы;
- Хотя и реализованна защита физической состовляющей канала передачи данных (благодаря 
  протоколу TCP), не организованна информационная защита.
- Возникает баг при экспорте отчета о более чем четырёх датчиках в формат Excel;
- Нет автоматических переходов между текстовыми полями по нажатии на клавишу Enter;
- Графическое представление графика потребления планировалось организовать в виде 
  гистограммы;
